# 🔑 JSON Web Token (JWT)

## Goal

Understand what a JSON Web Token (JWT) is, how it works internally, and why modern REST APIs use JWT for authentication.

This chapter explains JWT using the ShopSphere project.

---

# What is JWT?

JWT stands for

**JSON Web Token**

It is a compact, digitally signed token that allows a server to verify the identity of a user without storing session information.

JWT is commonly used in

- REST APIs
- Mobile Applications
- Microservices
- Single Page Applications (SPA)

---

# Why Do We Need JWT?

Imagine a customer logs into ShopSphere.

Without JWT

```
Login

↓

Server remembers user

↓

Every request

↓

Lookup Session
```

The server has to remember every logged-in user.

With JWT

```
Login

↓

JWT Generated

↓

Browser Stores Token

↓

Every Request Sends Token

↓

Server Verifies Token
```

The server remains stateless.

---

# Real World Analogy

Imagine entering an office building.

Reception verifies your identity.

↓

Gives you an ID card.

↓

You show the ID card whenever entering another floor.

Nobody verifies your identity again.

JWT works like that ID card.

# JWT Structure

A JWT has three parts.

```
Header

.

Payload

.

Signature
```

Example

```
xxxxx.yyyyy.zzzzz
```

Three parts.

Separated by dots.

---

# Header

The Header contains metadata.

Example

```json
{
    "alg":"HS256",
    "typ":"JWT"
}
```

Meaning

```
Algorithm

↓

HS256

Type

↓

JWT
```

---

# Payload

Payload contains claims.

Example

```json
{
    "customer_id":"550e...",

    "email":"maneesh@gmail.com",

    "role":"customer",

    "exp":1780000000
}
```

Notice

The payload is **not encrypted**.

It is only Base64 encoded.

Anyone can decode it.

Never store passwords here.

---

# Signature

The signature protects the token.

```
Header

+

Payload

+

Secret Key

↓

Hash Function

↓

Signature
```

If someone changes even one character,

the signature becomes invalid.

# Why Can't Users Modify JWT?

Imagine someone changes

```
role

↓

customer

↓

admin
```

The Payload changes.

↓

Signature no longer matches.

↓

Token rejected.

This is why JWTs are digitally signed.

---

# JWT Expiration

JWTs should expire.

Example

```
Access Token

15 Minutes
```

After expiry,

User must authenticate again or use a Refresh Token.

---

# Access Token vs Refresh Token

Access Token

- Short-lived
- Used for APIs

Refresh Token

- Long-lived
- Used to generate new Access Tokens

Typical Flow

```
Login

↓

Access Token

15 Minutes

↓

Expired

↓

Refresh Token

↓

New Access Token
```

# Where Should JWT Be Stored?

Options

### Local Storage

Easy

But vulnerable to XSS.

---

### Session Storage

Safer

But cleared when browser closes.

---

### HttpOnly Cookie

Most secure.

Cannot be accessed using JavaScript.

Preferred for production web applications.

# JWT Advantages

- Stateless
- Scalable
- Fast
- Easy to share across microservices
- No server-side session storage

---

# JWT Disadvantages

- Difficult to revoke before expiry
- Token size larger than Session IDs
- Payload is visible
- Secret key must be protected

---

# Best Practices

- Always use HTTPS
- Keep Access Tokens short-lived
- Never store passwords in JWT
- Protect signing secret
- Validate token on every protected request

---

# Common Mistakes

❌ Storing passwords inside JWT

❌ Using very long expiration times

❌ Forgetting to verify signatures

❌ Sending JWT over HTTP instead of HTTPS

---

# Interview Questions

### What is JWT?

A digitally signed token used to authenticate users in stateless applications.

---

### Is JWT encrypted?

No.

The Header and Payload are Base64 encoded.

The Signature protects against tampering.

---

### Why can't someone modify a JWT?

Changing the Header or Payload changes the Signature.

The server verifies the Signature before trusting the token.

---

### Why use JWT instead of Sessions?

JWT is stateless, making it easier to scale applications horizontally.

---

### What is the difference between Access Token and Refresh Token?

Access Token is used to access APIs and expires quickly.

Refresh Token is used to obtain a new Access Token without asking the user to log in again.
